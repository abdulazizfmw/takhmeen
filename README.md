# 🎯 لعبة التخمين بالصور

لعبة ويب (فرونت-اند فقط) — اختر تصنيفاً، تظهر لك صورة عشوائية عالية الجودة،
خمّن ذهنياً، اكشف الإجابة، ثم قيّم نفسك (صح/خطأ). مع عدّاد نقاط وأنيميشن ودعم RTL.

التصنيفات: **حيوان · فاكهة · علم دولة · أكلة سعودية · مدينة سعودية** (٢٠ صورة لكل تصنيف).

الصور تُخزَّن في **Cloudflare R2** وتُعرض عبر رابطها العام.

---

## التشغيل السريع (وضع تجريبي — بدون صور)

اللعبة تعمل فوراً بصور بديلة قبل ربط R2:

```bash
python3 -m http.server 4599
# افتح http://localhost:4599
```

عندما يكون `R2_PUBLIC_URL` فارغاً في `config.js`، تعمل اللعبة في «الوضع التجريبي»
(صور SVG بديلة تُظهر اسم العنصر) لاختبار كل شيء دون رفع صور.

---

## جلب الصور ورفعها إلى R2

الأداة تجلب صوراً حرّة من **ويكيبيديا/ويكيميديا كومنز**، تحسّنها (JPEG، عرض ≤ 1400px)،
ترفعها إلى R2، ثم تولّد `config.js` تلقائياً.

### ١) المتطلبات على R2 (مرة واحدة)
- أنشئ **bucket**.
- فعّل **Public Access** (رابط `*.r2.dev`) أو اربط **نطاقاً مخصّصاً**.
- أضف قاعدة **CORS** للسماح للمتصفح بجلب الصور (استبدل النطاق بنطاق موقعك، أو `*` للاختبار):

```json
[{ "AllowedOrigins": ["*"], "AllowedMethods": ["GET"], "AllowedHeaders": ["*"] }]
```

- أنشئ **R2 API Token** (S3) للحصول على Access Key / Secret.

### ٢) التثبيت والتشغيل (المسار الموصى به — خطوتان)

فصلنا «حلّ الروابط» عن «الرفع» لأنه أكثر متانة أمام حدود معدّل ويكيميديا:

```bash
cd tools
npm install

# (أ) حلّ روابط الصور الحرة → يكتب config.js بروابط مباشرة (لا يحتاج R2)
node resolve-urls.js

# (ب) رفع الصور إلى R2 وإعادة كتابة config.js ليقرأ من R2
R2_ACCOUNT_ID=xxxxxxxx \
R2_ACCESS_KEY_ID=xxxxxxxx \
R2_SECRET_ACCESS_KEY=xxxxxxxx \
R2_BUCKET=my-bucket \
R2_PREFIX=guess_game/ \
R2_PUBLIC_URL=https://pub-xxxx.r2.dev \
node upload-from-config.js
```

- `resolve-urls.js` يجلب الروابط عبر واجهة ويكيميديا فقط (سريع)، ويصلح لمعاينة اللعبة فوراً دون R2.
- `upload-from-config.js` يُنزّل الصور من الروابط المحلولة، يحسّنها، يرفعها إلى `R2_PREFIX`،
  ويعيد كتابة `config.js` ليشير إلى R2. أي صورة يتعذّر رفعها تبقى **باحتياطي** رابطها المباشر
  (تظهر في اللعبة)، وإعادة التشغيل ترفعها لاحقاً. **قابل لإعادة التشغيل** (يتخطّى الموجود عبر HeadObject).
- إن حظرت ويكيميديا معدّل التنزيل، أعد تشغيل الخطوة (ب) عدة مرات بفواصل زمنية — كل تمريرة تملأ ما بقي.
- **المفاتيح لا تُحفظ في أي ملف** ولا تُرفع؛ تبقى في البيئة فقط.

خيارات: `FORCE=1` لإعادة الرفع، `R2_PREFIX` لمجلد داخل الـ bucket، `MAX_WIDTH`.

> بديل بخطوة واحدة: `fetch-and-upload.js` يجلب ويرفع معاً، لكنه يستعلم من ويكيميديا أثناء الرفع
> فيكون أكثر عرضة لحدود المعدّل. المسار ذو الخطوتين أعلاه أفضل عملياً.

---

## النشر

ارفع الملفات الثابتة على أي استضافة (Cloudflare Pages / Netlify / GitHub Pages…):
`index.html`, `styles.css`, `app.js`, `config.js`. مجلد `tools/` ليس جزءاً من الموقع.

---

## البنية

| الملف | الوظيفة |
|---|---|
| `index.html` · `styles.css` · `app.js` | الواجهة والمنطق والأنيميشن |
| `config.js` | الرابط العام + أسماء الصور لكل تصنيف (يُولَّد تلقائياً) |
| `tools/subjects.js` | قوائم العناصر (٢٠/تصنيف) ومصادرها على ويكيميديا |
| `tools/resolve-urls.js` | حلّ روابط الصور الحرة → معاينة فورية دون R2 |
| `tools/upload-from-config.js` | رفع الصور إلى R2 + إعادة كتابة config.js (المسار الموصى به) |
| `tools/fetch-and-upload.js` | بديل بخطوة واحدة (جلب + رفع معاً) |

## اختصارات لوحة المفاتيح (أثناء اللعب)
`مسافة/Enter` كشف الإجابة · `→/C` صح · `←/X` خطأ · `N` التالي.
